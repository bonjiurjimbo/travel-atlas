# 旅の台帳 編集方法

このフォルダを1つの旅行ドキュメントとして管理します。

## ファイル構成

- `index.html`: 表示用のページです。ふだん編集しなくて大丈夫です。
- `trips-data.js`: 旅行・日別旅程・準備リスト・費用・メモを編集するファイルです。

## 旅程を編集する場所

`trips-data.js` の中にある `window.TRIPS_DATA = [` から `];` までが旅程データです。

1つの旅行は `{ ... }` のまとまりです。旅行を追加するときは、既存の旅行ブロックをコピーして、末尾に貼り付けてから中身を書き換えます。

## 主な項目

- `id`: 英数字とハイフンで付ける旅行IDです。例: `tokyo-singapore-overland`
- `title`: 旅行名です。
- `year`: 予定年です。
- `season`: 時期です。
- `status`: `plan`, `ready`, `live` のどれかです。
- `region`: 地域です。トップページの絞り込みに使います。
- `length`: 旅行日数です。
- `countries`: 国・地域の一覧です。
- `route`: ルート概要です。
- `concept`: この旅で何を見たいか、何を確かめたいかです。
- `prep`: 出発前チェックリストです。
- `packing`: 持ち物リストです。
- `budget`: 費用表です。
- `notes`: 調べたこと、残課題、旅の方針です。
- `days`: 日別の旅程です。

## 日別旅程の書き方

`days` の中に日ごとの予定を書きます。

```js
{
  label: "1日目",
  date: "東京から西へ",
  aim: "この日の狙いを書く",
  checks: ["乗車券", "天気", "現金"],
  schedule: [
    ["07:00", "出発", "東京駅", "予約確認", "23,000円"],
    ["12:30", "昼食", "博多駅周辺", "水を買う", "2,000円"]
  ]
}
```

`schedule` は左から順に、時刻、予定、場所、予約・確認、費用です。

## iPhone/iPadで見る方法

見るだけなら、このフォルダをWebに置くのが一番安定します。

- GitHub Pages
- Netlify
- Cloudflare Pages

まずは `travel-atlas` フォルダごと公開します。公開後は、iPhone/iPadのSafariでURLを開けば見られます。

## iPhone/iPadで編集する方法

最初の現実的な方法は2つです。

1. GitHub上で `trips-data.js` を編集する
2. Google Sheetsに旅程を置き、後でこの形式に変換する

旅行中も編集するなら、Google Sheets方式が使いやすいです。計画・準備・現地修正はGoogle Sheetsで行い、表示用ページはそこから作る構成にできます。

## Codexに頼むときの例

「`outputs/travel-atlas/trips-data.js` に、2028年初夏のバルト三国旅行を追加してください」

「東京からシンガポール旅の2日目に、午前中の両替とSIM設定を追加してください」

「この旅行を旅行中ステータスに変更してください」
